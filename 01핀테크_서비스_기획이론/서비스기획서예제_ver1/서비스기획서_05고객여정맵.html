개요 페르소나 문제 정의 핵심 가치 고객 여정맵 IA정보구조도 주요기능 요구사항 와이어프레임 구현 계획

고객 여정맵(Customer Journey Map) 이론

고객 여정맵은 사용자가 서비스에 접촉하고, 탐색하고, 참여하고, 유지·이탈하기까지 전 과정에서 겪는 경험(EQ)과 행동(BQ)을 시각화한 분석 도구이다.

  • 전체 사용자 흐름을 단계별로 파악하여 Pain Point → 해결 기능을 연결
  • 페르소나별 행동·감정 패턴을 반영해 정교한 UX 전략 수립에 도움
  • 온보딩/자산 연동/피드 추천/라이브 참여 등 핵심 전환 지점을 설계하는 기준
  • IA 설계, 알림 정책, 콘텐츠 전략과 직접 연결되는 핵심 문서

이 여정맵은 페르소나 3명(초보 탐색형·DIY 투자형·생활밀착형)의 니즈와 문제 정의(초보 난이도·자산 분산·신뢰 부족·과잉 정보)를 반영하여 EasyFin V2.0 Full 서비스 기준으로 설계되었다.

7. 고객 여정맵 (7-1. 단계별 요약)

본 고객 여정맵은 EasyFin V2.0의 Full 버전(정식 서비스 구조)을 기준으로 작성되었습니다.
※ 현재 MVP에서는 자산 연동/라이브 일부 기능이 제외될 수 있으며,
실제 MVP 플로우는 온보딩 → 홈 대시보드 탐색 → 마켓/커뮤니티 이용에 우선 집중합니다.

7-1. 단계별 요약

단계 행동(BQ) 인식/감정(EQ) Pain Point (데이터 기반) 기회 요소 (Value Proposition 연결)
1) 인지 SNS·콘텐츠·지인을 통해 서비스 최초 인지 “새로운 금융 서비스? 나에게 맞을까?” - 광고성 금융 정보 피로 증가
- 신뢰 기반 정보인지 판단 어려움
- 숏폼 기반 소비 증가로 짧은 관심 유지 		- 전문가/셀러 검증 강조 메시지
- 초보자·중급자별 가치 제안 분리 노출
- 숏폼 중심의 3초 Hook 콘텐츠 제공
2) 설치 & 가입 앱 다운로드 → 간편가입 → 서비스 소개 화면 확인 “가입이 너무 복잡하진 않겠지?” - 초보자: 설정·용어 부담
- 생활형: 개인정보 제공 불안
- 복잡한 온보딩/과한 입력 단계 		- 카드형 온보딩 + 쉬운 설명
- 용어 툴팁 지원(Easy Value)
- '왜 정보를 요구하는가' 투명 설명(Trust)
3) 자산 연동 금융기관 선택 → 인증 → 계좌·자산 연결 “보안 걱정도 되고, 실패하면 어떡하지?” - 연동 실패 경험(특히 중급 투자자)
- 개인정보 제공 불안
- 여러 앱을 쓰던 사용자에게 익숙하지 않은 플로우 		- 단계별 시각화·가이드(Trust)
- 연동 실패 자동 복구/재시도 UX
- 연동 완료 후 ‘내 자산 한눈에’ 즉시 경험 제공(Easy)
4) 탐색 대시보드 → 마켓 → 커뮤니티 탐색 “내 상황에 맞는 게 바로 보였으면…” - 정보 과다로 첫 탐색 난이도 높음
- 초보자: 어디서 시작해야 할지 모름
- 중급자: 포트폴리오 인사이트 부족 		- 개인 맞춤 피드(Intelligence)
- AI 기반 ‘지금 나에게 필요한 3가지’ 추천
- 가이드 태스크 플로 제공(Easy)
5) 참여 라이브 시청·채팅·질문·커뮤니티 활동 “실시간으로 묻고 답하니 더 재미있네!” - 질문 미응답(일방향 방송)
- 커뮤니티 품질 관리 어려움
- 정보의 신뢰도 편차 		- Q&A 고정·답변 하이라이트(Fun)
- 전문가 인증 배지 시스템(Trust)
- 댓글 요약·하이라이트 자동 생성(Intelligence)
6) 유지 푸시 알림 → 반복 방문 → 서비스 루틴 형성 “계속 쓰면 생활이 더 편해지겠는데?” - 유입은 되지만 루틴화 어려움
- 생활형 사용자의 지출·납입 관리 니즈 미충족 시 이탈
- 피드백 반영을 체감하기 어려움 		- 맞춤 알림(지출/포트폴리오/라이브)(Intelligence)
- 개선 로그 노출(Trust)
- 반복 방문 미션·커뮤니티 성장 요소(Fun)
