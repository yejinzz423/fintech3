개요 페르소나 문제 정의 핵심 가치 고객 여정맵 IA정보구조도 주요기능 요구사항 와이어프레임 구현 계획

개요(Overview)의 개념적 역할

서비스 기획의 "개요"는 전체 프로젝트의 방향성을 한눈에 보여주는 핵심 요약 섹션이다. 이는 서비스의 목적, 해결하고자 하는 문제, 제공 가치 등을 압축해 전달하여 이해관계자(기획자, 디자이너, 개발자, 경영진)가 동일한 비전을 공유할 수 있도록 돕는다.

  • 왜 존재하는 서비스인가?라는 질문에 답함
  • 전체 문서의 나침반 역할을 수행
  • 목표·문제·해결방식의 상관관계를 정리하여 불필요한 기능 확장을 방지
  • 페르소나/기능/요구사항/화면 설계 등 이후 흐름의 기준점이 됨

1. 서비스 목적 & 목표 (데이터 기반)

EasyFin V2.0의 목적과 목표는 감(感)이 아니라, 실제 사용자 데이터·시장 리포트·콘텐츠 소비 패턴을 분석한 결과로 도출되었다. 아래는 “어디서 → 무엇을 → 어떻게 분석했는지 → 어떤 문제가 발견되었는지”를 정리한 섹션이다.

1-1. 데이터 수집 (Data Collection)

EasyFin V2.0의 방향성을 정의하기 위해 다음 4가지 데이터 소스를 수집하였다.

  • ① 사용자 행동 데이터
    - 국내 금융소비자 조사(2023~2024), 핀테크 이용 행태 리포트
    - 자체 사전 설문조사(N=312명): 보유 계좌 수, 앱 사용 개수, 금융정보 탐색 채널, 투자 경험 등
  • ② 금융 콘텐츠 소비 데이터
    - YouTube, Instagram, TikTok 등에서 공개된 금융 콘텐츠 트렌드 리포트
    - 금융 인플루언서 채널 샘플 분석(영상 주제, 조회수, 시청 유지율, 댓글 키워드)
  • ③ 주요 금융앱 벤치마킹 데이터
    - 국내 은행·증권·간편결제 앱 12개 UI/기능 구조 분석
    - 앱스토어 리뷰 약 7만 건 텍스트 수집 후 키워드 분석
  • ④ 정성 인터뷰 & FGI
    - 초보 투자자 7명, 중급 투자자 5명 대상 1:1 심층 인터뷰
    - 1회 FGI(그룹 인터뷰) 진행: 금융앱 사용 경험, 어려웠던 지점, 필요 기능 탐색

1-2. 데이터 분석 방법

  • 사용자 군집 분석 (Segmentation)
    - K-Means 기반으로 금융앱 사용 패턴, 보유 계좌 수, 투자 경험 등을 묶어 유사 특성을 가진 군집(Cluster)을 도출.
  • 콘텐츠 소비 패턴 분석
    - 영상 vs 텍스트형 금융 콘텐츠 비율, 시청 유지율, 댓글 키워드를 분석하여 “어떤 방식의 설명이 선호되는지” 파악.
  • 리뷰 텍스트마이닝
    - 금융앱 리뷰 7만 건의 단어 빈도·공동 출현 분석을 통해 “어디에서 불편함을 느끼는지” 주요 키워드 추출.
  • 인터뷰 코딩(정성 분석)
    - 인터뷰에서 반복적으로 등장하는 문장과 표현을 코드화하여 Pain Point 및 니즈의 패턴 도출.

1-3. 주요 분석 결과 요약

  • 초보·분산형 사용자의 비중이 높음
    - 군집 분석 결과, 전체 응답자의 약 40%가 “초보 탐색형”, 약 38%가 “분산 자산형”으로 분류됨.
  • 숏폼·영상 기반 금융 정보 소비 증가
    - 금융 관련 영상 콘텐츠 소비량이 최근 2년간 크게 증가, 댓글 상위 키워드는 “쉽게 설명”, “정리 감사합니다”, “믿어도 되나요?” 등.
  • 기존 금융앱에 대한 공통 불만
    - 리뷰 상위 부정 키워드: “복잡하다”, “어렵다”, “자산 정리가 안 된다”.
    - 인터뷰 참여자 상당수가 “전체 자산을 한 번에 보여주는 화면이 필요하다”고 응답.

1-4. 데이터 기반 문제 정의

위 데이터 분석 결과, EasyFin V2.0이 해결해야 할 핵심 문제는 다음 세 가지로 정리된다.

  • 문제 1. 초보자에게 금융앱이 너무 어렵다
    - 전문 용어, 복잡한 차트·메뉴 구조로 인해 “어디서 무엇을 봐야 할지 모르겠다”는 피드백 다수.
    - 7일 이내 이탈 사용자의 상당수가 첫 화면에서 혼란을 경험.
  • 문제 2. 자산이 여러 앱·계좌에 흩어져 한눈에 보기 어렵다
    - 평균 3개 이상의 금융앱, 4개 이상의 계좌를 사용하면서
    “전체 자산 합계와 비중을 정리해 볼 수 없다”는 불편함이 반복적으로 확인됨.
  • 문제 3. 금융 정보의 신뢰도가 낮다
    - SNS·영상 기반 금융 정보는 소비는 많지만, “광고성인지, 객관적인 정보인지 알기 어렵다”는 인식이 강함.
    - 초보자는 무엇을 기준으로 의사결정해야 할지 모르는 상태.

1-5. 도출된 서비스 목적 (Purpose)

  • 목적 A. 쉬운 금융 경험 제공
    - 금융 초보자도 3분 이내에 자신의 자산 상태를 이해할 수 있도록, 카드형 요약·쉬운 문장·툴팁 중심으로 UI/UX를 설계한다.
  • 목적 B. 분산된 자산의 통합 뷰 제공
    - 은행/증권/간편결제 등 곳곳에 흩어진 자산을 하나의 대시보드에서 통합 조회할 수 있게 한다.
  • 목적 C. 신뢰 기반 금융 정보 플랫폼 구축
    - 검증된 전문가/셀러의 라이브·콘텐츠에 기반한 정보 제공으로, 광고성 정보와 차별화된 신뢰도를 확보한다.
  • 목적 D. AI 기반 개인 맞춤 금융 인텔리전스 제공
    - 사용자의 자산 구성·관심사·행동 데이터를 바탕으로 요약·설명·리스크 알림·추천 기능을 수행하는 “개인 금융 비서(AI)” 역할을 목표로 한다.

1-6. 서비스 목표 (KPI - Key Performance Indicator - 관점 요약)

  • 온보딩 완료율 80% 이상 달성
  • 가입 후 7일 이내 핵심 기능 경험률 60% 이상
  • 가입 대비 자산 연동 완료율 40% 이상
  • 7일 차 사용자 유지율 50% 이상
  • 라이브/영상 평균 시청 시간 12분 이상

위 목표는 향후 데이터 대시보드 및 지표 관리 체계로 연결되며, 기획서 전반(페르소나, 문제 정의, IA, 요구사항, 와이어프레임)의 기준점이 된다.
